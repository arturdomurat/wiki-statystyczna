R — poradnik dla początkujących

Kiedy tego potrzebujesz

Zaczynasz pracę z R i potrzebujesz szybkiego startu: instalacja, podstawy, najważniejsze pakiety do analiz statystycznych w psychologii. Albo używasz SPSS, ale chcesz przejść na R, bo jest darmowy, elastyczny i coraz bardziej wymagany przez czasopisma.

Dlaczego R?

  • Darmowy i open source — bez licencji instytucjonalnej.
  • Reprodukowalność — skrypt R dokumentuje dokładnie, co zrobiłeś. W SPSS klikasz w menu i nie masz śladu.
  • Elastyczność — od prostych t-testów po SEM, modele mieszane, meta-analizy, Bayesian.
  • Społeczność — tysiące pakietów, Stack Overflow, kursy online.
  • Wykresy — ggplot2 produkuje wykresy gotowe do publikacji.

Instalacja

  1. R — pobierz z https://cran.r-project.org (wybierz system operacyjny).
  2. RStudio — IDE do pracy z R. Pobierz z https://posit.co/download/rstudio-desktop/. RStudio to „nakładka" na R z edytorem, konsolą, podglądem wykresów i pomocą.

Podstawy składni

# Przypisanie wartości
x <- 5
nazwa <- "tekst"

# Wektor
wyniki <- c(12, 15, 18, 22, 9)
mean(wyniki)   # średnia
sd(wyniki)     # odchylenie standardowe

# Wczytanie danych z CSV
dane <- read.csv("moje_dane.csv", header = TRUE, sep = ";")

# Podgląd danych
head(dane)     # pierwsze 6 wierszy
str(dane)      # struktura (typy zmiennych)
summary(dane)  # statystyki opisowe

Typowe analizy — szybki start

Test t dla prób niezależnych

t.test(DV ~ grupa, data = dane, var.equal = FALSE)  # Welch (domyślny!)
library(effectsize)
cohens_d(DV ~ grupa, data = dane)

ANOVA jednoczynnikowa

library(car)
model <- aov(DV ~ grupa, data = dane)
Anova(model, type = 3)
library(effectsize)
eta_squared(model, partial = TRUE)

# Post hoc
TukeyHSD(model)

Korelacja

cor.test(dane$x, dane$y, method = "pearson")
cor.test(dane$x, dane$y, method = "spearman")

Regresja

model <- lm(DV ~ predyktor1 + predyktor2, data = dane)
summary(model)
library(car)
vif(model)  # współliniowość

Rzetelność

library(psych)
alpha(dane[, c("item1", "item2", "item3", "item4")])
omega(dane[, c("item1", "item2", "item3", "item4")])

Dobre praktyki

  1. Używaj projektów RStudio (.Rproj) — trzymaj dane, skrypty i wyniki w jednym folderze.
  2. Komentuj kod — # ten komentarz wyjaśnia, co robię.
  3. Nie modyfikuj oryginalnych danych — wczytaj surowe dane, przetwórz w skrypcie, zapisz wyniki osobno.
  4. Zapisuj skrypt, nie wyniki w konsoli — skrypt jest reprodukowalny, konsola nie.
  5. Nazwy zmiennych — po angielsku, bez spacji i polskich znaków (lęk → lek lub anxiety).

Częste błędy początkujących

  1. Zapominanie o library() na początku sesji. Zainstalowanie pakietu to jedno, załadowanie to drugie.

  2. Błąd object not found — literówka w nazwie zmiennej, niezaładowane dane, lub zmienna w cudzysłowie zamiast bez (i odwrotnie).

  3. Separator dziesiętny — R używa kropki (3.14), nie przecinka (3,14). Przy wczytywaniu polskich CSV: read.csv2() lub read.csv(..., dec = ",").

  4. Czynniki (factors) zamiast liczb — R czasem traktuje kolumnę liczbową jako factor. Sprawdź str(dane) i konwertuj: as.numeric(as.character(zmienna)).

  5. Kopiowanie kodu z PDF/Worda — znaki cudzysłowu „" zamiast "" powodują błędy. Pisz kod w RStudio.
